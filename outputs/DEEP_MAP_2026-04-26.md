## 2. Critical flows

### 2.1 Track-change (the trunk)

```
User taps Play / Next → playerStore.setCurrentTrack(track)
  ├─ playbackState transition (idle → loading | playing → loading)
  ├─ history.push, queue.shift, addToHistory
  └─ AudioPlayer's [currentTrack.trackId] effect fires:
       1. cancel previous AbortController + bump changeToken
       2. r2KnownStore.has(id)?
          ├─ YES → el.src = R2 URL (fast path, ~0ms perceived)
          └─ NO  → engageSilentWav (so audio element keeps "playing")
                   r2HasTrack(id) HEAD probe (4s timeout)
                   ├─ hit  → el.src = R2 URL, markR2Known
                   └─ miss → setSource('iframe') — loop silent WAV; iframe carries audio
       3. logPlaybackEvent('play_start' | 'play_fail')
       4. handleCanPlay → fadeInMasterGain
```

**Where it can go wrong silently:**
- `setSource('iframe')` while `playbackState === 'idle'` → ALLOWED (added in v412 fix); without it `state_illegal` fires and silent WAV is rejected.
- AbortController on the in-flight probe is the only thing protecting against rapid skip writes — `changeToken` is the cross-check.
- r2Probe HEAD with `_v=Date.now()` defeats edge cache (audit-2 P2). Every probe hits origin.

### 2.2 Hot-swap (iframe → R2 mid-track)

```
playbackSource === 'iframe' AND duration === 0 OR duration >= 300
  → useHotSwap mounts (was DEAD code before v644)
     1. supabase realtime subscribe to voyo_uploads_extracted (WHERE youtube_id=...)
     2. parallel poll r2Probe every 2s (capped at 30s)
     3. on either signal hit:
        - el2 = new Audio(R2 URL); el2.preload = 'auto'
        - await canplay (2.5s timeout — never cancelled, audit-2 P2)
        - 2s equal-power crossfade (gain ramp on both elements)
        - el.src = R2; setSource('r2'); r2KnownStore.add(id)
        - el.volume = 1.0 (was storeVol → 30% drop bug, fixed v651)
```

**Trip wires:**
- Supabase channel registry: `removeChannel()` is required, not just `unsubscribe()` (audit-2 P0-RT-3 — fixed in `useHotSwap` cleanup, but lib/realtime/reconnect.ts is the leaky one).
- Volume contract: enhanced chain says HTML=1.0, masterGain attenuates. Hot-swap must respect.

### 2.3 OYE tap (the contribution ritual)

```
TrackCard double-tap or OyeButton tap → app.oyeAndBoost(track)
  → oyeCommit(track, opts) inside services/oyo/app.ts
     1. logPlaybackEvent('trace', { subtype: 'oye_commit' })
     2. boostTrack (downloadStore) — downloads to IndexedDB, gates R2 extraction
     3. addToQueue (playerStore, position 0 = next up)
     4. setExplicitLike (preferenceStore — persists)
     5. markWarming (warmingStore) — purple "cooking" pill 60s safety timer
     6. addReaction → realtime broadcast voyo_reactions
```

**Narralogy** (computeOyeState in OyeButton.tsx):
- **Orange** = added (warming, not in R2 yet)
- **Purple** = cooking (iframe-as-audio engaged for this track)
- **Gold-faded** = in Disco (R2-cached, not yet Oye'd by user)
- **Gold-filled** = Oye'd + landed

### 2.4 Search → tap non-R2 result

```
User types → 120ms debounce → /search/?q=...
results render → /exists/{id} batch (audit-2 P1 — uncapped concurrency, no AbortController in v651 fix only addresses cancellation)
User taps non-R2 row → handleSelectTrack:
  1. ensureTrackReady(id, p=10) — kicks R2 extraction at high priority
  2. app.oyeCommit(track) — see flow 2.3
  3. markWarming(id) — purple pill
  4. showToast({ type: 'warming', ... }) — "cooking…" message
  5. morphTimerRef = setTimeout(3000, () => showToast({ type: 'play_now' }))
  6. announcedDiscoRef tracks ids that have lit "in_disco" celebration
```

**v656** added the patience-as-contribution celebration — when warmingSet ∩ r2KnownSet hits, the row pulses gold with "✦ in Disco" pill.

### 2.5 Mini player → Take Out portal (v663-v678)

```
videoTarget = 'portrait' (mini player up)
ExpandVideoButton phase machine (in card top-right):
  'mini' (purple) → tap = enter mini player
       ↓ 5s
  'morphing' (purple→orange pulse, 0.9s)
       ↓
  'takeout' (orange) → tap = pipService.enter()
       ↓ 2.2s
  'parked' — chip fades to opacity:0 from card spot

When portalProgress > 0.2 (user scrolling toward mix board):
  BottomTakeOutChip rises from bottom-right
  ↓ 5s settled
  decays to 7% opacity (still tappable)

DRAG-TO-PORTAL (v677-678):
  User grabs floating mini iframe → drags right
  portalGlow = clamp((120 - distFromRightEdge) / 100, 0, 1)
  Right edge: thin orange seam (3px → 6px when armed)
  + 120px halo + 260px screen-blend interior wash + vertical "TAKE OUT" label
  Haptics: selection on glow≥0.3, light on armed (0.85), success on drop
  Drop while armed → pipService.enter() + setVideoTarget('hidden')
```

**pipService canvas-composite implementation** (v661):
- `useMiniPiP` hook in AudioPlayer creates 360² canvas, paints VOYO chrome
- canvas.captureStream(30fps) → off-screen <video> → requestPictureInPicture
- Cross-origin YT iframe blocks native PiP — this is the only path for a PWA over a YT iframe.

### 2.6 Error recovery (two-tier boundaries)

```
AppErrorBoundary (App.tsx:137)
  └─ getDerivedStateFromError → criticalError() to logger
     componentDidCatch → state.hasError = true
     Recovery ladder:
       1st crash → soft reload (5s delay, preserves caches)
       2nd crash within 20s → still soft reload
       3rd crash within 20s → NUKE: localStorage.clear() + caches.delete() + hard reload
     Counter persisted in sessionStorage `voyo-crash-counter-v1`
     Boot success marker `voyo-boot-ok-v1`

AudioErrorBoundary (audio/AudioErrorBoundary.tsx)
  componentDidCatch → trace('audio_boundary_catch')
     Crash-loop guard: 3 catches in 5s → halt remount, render null
     teardownAudioChain() before remount
     Remount via state-toggle (key change forces fresh AudioPlayer mount)
```

**The gap:** Neither boundary catches `unhandledrejection`. Async `await` chains throwing outside React's tree go to /dev/null.

### 2.7 Background recovery

```
visibilitychange / focus / pageshow → AudioContext resume ladder
  audioEngine.ts:75-78: trace('ae_resume_attempt') → resume → ok/rejected
  bgEngine.ts: heartbeat every 4s while playing
    if ctx.state ∈ {suspended, interrupted, closed} → resume + el.play()
    if ctx.state === 'closed' → teardownAudioChain (full rebuild)
  WakeLock NOT implemented (audit-2 noted) — tab is suspend-eligible
  Silent WAV keeper preserves audio focus during BG (bgEngine.ts:149)
```

### 2.8 Scroll to mix board (Layer B)

```
User scrolls main canvas → handleHeaderScroll → rAF → setPortalProgress
portalProgress: 0 (home, Layer A card centered) → 1 (full canvas, Layer B faded)

Layer B opacity (VoyoPortraitPlayer.tsx:5481):
  portalProgress < 0.55: 1 - p * 0.7  (1.0 → 0.6)
  portalProgress >= 0.55: max(0, 0.6 - (p - 0.55) * 1.5)  (0.6 → 0)

White-flash bug (v672 fix): filter:blur(${p * 8}px) was re-rasterizing on Safari
  → dropped, opacity carries the fade
v673 fix: backdrop-blur-2xl on Layer B was recompositing every frame
  → dropped, gradient is 92-99% opaque so blur was invisible anyway
```

## 4. Telemetry surface (current state)

### What we instrument well

**Schema** (`supabase/migrations/telemetry.sql`):
- `voyo_playback_events` table: id, created_at, event_type, track_id, track_title/artist, source, error_code, latency_ms, is_background, user_agent, session_id, meta (JSONB)
- 4 indices: created_at DESC, (event_type, created_at) DESC, track_id, error_code (partial)
- RLS: anon insert-only. Read requires service key.
- View: `voyo_recent_failures` — 1-min bucketed play_fail counts in last hour

**Client batcher** (`services/telemetry.ts`):
- 10s flush OR 20-event buffer flush, hard cap 100 events
- Foreground: fetch with keepalive
- Background: `navigator.sendBeacon` (the ONE API not throttled in BG)
- pagehide listener + visibilitychange→hidden listener for unload survival
- Auto-disable after 10 consecutive failures (no spam)

### What we don't instrument (the gaps)

| Gap | Surface area | Severity |
|---|---|---|
| `window.onerror` listener | All sync throws outside React tree | **HIGH** |
| `unhandledrejection` listener | Every uncaught async (`async () => { throw }`) | **HIGH** |
| `services/api.ts` catch blocks (14 sites) | Edge timeouts, R2 misses, search 5xx — silent degradation | **HIGH** |
| `services/voyoStream.ts` catch blocks (5 sites) | Ensure/queue failures swallowed | MEDIUM |
| `lib/voyo-api.ts` (1 site) | Voyo profile API failures | LOW |
| `lib/dahub/dahub-api.ts` (14 sites) | Universe, friends, presence | MEDIUM |
| Service worker fetch/install/activate failures | Cache failures invisible | LOW |
| Web Vitals (LCP, INP, CLS) | No real-device perf data | MEDIUM |
| User-visible error UI | Network glitches just disappear | MEDIUM |

**Dead enum values** (defined in `PlaybackEventType` but never emitted as `event_type`):
- `'stream_ended'` — would catch unexpected ends mid-session
- `'bg_disconnect'` / `'bg_reconnect'` — bgEngine fires the *traces* equivalent but not the typed event_type, so the dashboards don't catch them

### Failure-surface arithmetic

- 532 `catch` blocks total in src/
- 81 telemetry call sites
- Telemetry-to-catch ratio ≈ 15%. Most catches are silent (haptics, storage, dom mutation, where silence is correct). The 14 in api.ts and 5 in voyoStream.ts are where it matters.

## 7. Proposed observability hardening (the plan)

Three waves, each surgical. None are observe-and-adjust violations because telemetry IS what observe-and-adjust requires.

### Wave 1 — Catch the silent failures (highest ROI, ~80 LOC)

**1A. Window-level capture** (`utils/errorReporter.ts`, new ~30 LOC)
- `window.addEventListener('error', e => trace('window_error', null, { msg, src, line, col, stack: e.error?.stack?.slice(0, 500) }))`
- `window.addEventListener('unhandledrejection', e => trace('unhandled_rejection', null, { reason: String(e.reason).slice(0, 200), stack: e.reason?.stack?.slice(0, 500) }))`
- Mounted once in `main.tsx` before React.

**1B. Wire api.ts catch blocks** (14 sites, ~14 LOC)
- Each catch: `trace('api_fail', null, { endpoint: 'edge_extract' | 'r2_check' | 'search' | ..., status, msg })`
- No retry logic added — just observability.

**1C. Wire 5 voyoStream catches + 14 dahub-api catches** (~19 LOC)
- Same pattern, distinct subtypes.

**1D. Resurrect dead enum values** (~3 LOC across bgEngine + AudioPlayer)
- Replace 3 traces with proper event_type so they show in the failures view: `bg_disconnect`, `bg_reconnect`, `stream_ended`.

**Expected outcome:** ~3x today's telemetry signal volume, but every previously-silent failure becomes queryable in `voyo_playback_events`.

### Wave 2 — Web Vitals + a tiny dashboard (~50 LOC)

**2A. Web Vitals** (new `services/webVitals.ts`, ~25 LOC)
- Add `web-vitals` package (4kb gzip)
- Report LCP, INP, CLS via existing `logPlaybackEvent` as `event_type: 'vital'` with `meta: { metric, value, rating }`.
- Fires once per session lifecycle, not per render.

**2B. Dev-only telemetry view** (new route or `/__telemetry` HTML, ~25 LOC)
- Service-key-gated read of `voyo_recent_failures` view + last 50 events for current session
- Only mounts when `localStorage.voyoDebug === '1'`
- Could be a static HTML in `public/` that the dev opens directly with the service key in URL hash.

### Wave 3 — Spike alerting (deferred until we see real-world signal)

- Edge function on Supabase that runs every 5 min, queries `voyo_recent_failures`, posts to a Discord/email webhook if `count > N`.
- Punt on this until Wave 1 has been live for a week and we know what "normal" looks like.

### What we explicitly skip

- **Sentry/PostHog/LogRocket.** Supabase is already the storage layer and the cost+complexity of a third-party doesn't beat extending what we have. Reconsider only if we want session replays.
- **Crash percentages.** We have a counter in sessionStorage; persist it to telemetry on next boot, that's enough.
- **Full perf-trace timeline.** Web Vitals covers 95% of what we'd want.

## 8. Quick reference — testing the app while observing

**While developing:**
```js
// Console:
window.voyoTelemetry.enableDebug()      // (currently no-op but keeps the flag set)
window.voyoTelemetry.stats()            // see buffer, session, failure count
window.voyoTelemetry.flush()            // force flush
window.voyoTelemetry.sessionId()        // grab session id for queries
```

**Querying the database (need service key):**
```sql
-- Last hour failures
SELECT * FROM voyo_recent_failures;

-- Full trace for a session
SELECT created_at, event_type, track_id, error_code, meta
FROM voyo_playback_events
WHERE session_id = '<id>'
ORDER BY created_at;

-- Most-failing tracks last 24h
SELECT track_id, track_title, COUNT(*) as fails
FROM voyo_playback_events
WHERE event_type = 'play_fail' AND created_at > NOW() - INTERVAL '24 hours'
GROUP BY 1, 2 ORDER BY 3 DESC LIMIT 20;
```

**Flow-test cheat sheet:**
- Track-change race → tap two non-R2 results within 1.5s
- Hot-swap → search non-R2, wait ~11s for crossfade
- OYE narralogy → tap card → orange → cooking → gold
- Take-out portal → drag floating mini iframe right → orange seam glows → drop into portal
- Background → lock screen during playback → unlock → audio still playing
- Crash recovery → throw inside a render → soft reload, counter increments, hits 3 in 20s = nuke
